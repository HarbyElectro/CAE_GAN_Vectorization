# Latent Vector Format

This document describes the latent-vector formats used in `CAE_GAN_Vectorization`.

Latent vectors are compressed learned representations generated by the encoder part of an autoencoder.

They can be used for:

- Compression
- Reconstruction
- Similarity search
- Semantic retrieval
- Multimedia indexing
- Storage optimization

---

## CAE Latent Vectors

Script:

```bash
CAE_CelebA.py
```

Typical output:

```bash
runs/celeba_cae/celeba_encoded_images.npy
```

Expected format:

```bash
(N, latent_dim)
```

Example:

```bash
(10000, 64)
```

Where:

- `N` is the number of encoded images.
- `latent_dim` is the dense bottleneck dimension.

---

## GAN-AE Latent Vectors

Script:

```bash
Gan_autoencoder_Cfp.py
```

Typical output:

```bash
runs/gan_ae_cfpd/latents.npy
```

Expected format:

```bash
(N, C_lat, H_lat, W_lat)
```

Where:

- `N` is the number of encoded images.
- `C_lat` is the number of latent channels.
- `H_lat` is the latent height.
- `W_lat` is the latent width.

Example:

```bash
(5000, 16, 8, 8)
```

---

## HDF5 Latent Storage

The GAN-AE script can optionally store latents in HDF5 format.

Example output:

```bash
runs/flowers_latent.h5
```

Example command:

```bash
python Gan_autoencoder_Cfp.py \
  --dataset_type image_folder \
  --data data/flowers \
  --out runs/gan_ae_flowers \
  --latents_path runs/flowers_latent.h5 \
  --store_images_in_h5 \
  --h5_gzip 6
```

Benefits of HDF5:

- Efficient large-scale storage
- Optional compression
- Structured arrays
- Good for research experiments with many samples

---

## Recommended Metadata

Each latent export should be accompanied by metadata such as:

```json
{
  "model": "gan_autoencoder",
  "dataset": "CFPD",
  "latent_shape": [5000, 16, 8, 8],
  "image_size": 64,
  "latent_dtype": "float16",
  "checkpoint": "runs/gan_ae_cfpd/ckpt_epoch050.pt"
}
```

## Loading Latents in Python

```python
import numpy as np

latents = np.load("runs/gan_ae_cfpd/latents.npy")
print(latents.shape)
```

---

## Flattening Latents for Retrieval

Some retrieval algorithms require 2D input:

```python
latents_flat = latents.reshape(latents.shape[0], -1)
```

This transforms:

```bash
(N, C_lat, H_lat, W_lat)
```

into:

```bash
(N, C_lat * H_lat * W_lat)
```
